# 通用架构 × 专业能力

这本书从开篇到现在，反复出现同一个命题：**专用 Agent = 通用 Agent 架构 × 专业业务能力。** 这最后一章之一，正面拆这个乘法——两个因子各是什么、它们在哪里相乘，以及一个我认为最重要的判断：**壁垒，到底在乘号的哪一边。**

## 「通用」的那一半：可迁移的架构

先看「通用」这个因子。它包括这本书前两部分、以及第六部分讲的几乎所有东西：

- 模型驱动的循环、工具即能力边界、上下文工程、记忆、子 Agent（第一部分）；
- 模型分层、网关与 fallback、并发闸门、弱模型护栏（第二部分）；
- 部署、性能、并发、调试、监控（第六部分）。

这些的共同特点是——**换个领域，它们基本不变。** 你做一个医疗 Agent、法律 Agent、金融 Agent，这套骨架都成立：都要一个循环、都要工具、都要管上下文和记忆、都要选模型架护栏、都要部署和监控。

正因为它通用，它也**可以学、可以抄**。Claude Code 这类成熟范式，已经把这套骨架趟出来了。你不需要重新发明它——你需要的是理解它、然后把它搬到你的领域。

## 「专业」的那一半：啃不动的领域

再看「专业」这个因子。它是这本书第三、四、五、七部分的核心：

- 领域的动作词典、工具设计、真实数据源、数据护城河（第三部分）；
- 用户到底要什么、领域独有的需求（自驾的驾时、多人的共识）、领域知识建模（第四部分）；
- 真实性的铁律、境内外的差异（第五部分）；
- 这个领域怎么变成一个产品、一个平台（第七部分）。

这些的共同特点是——**没有现成答案，每个领域都得自己啃。** 「一条好的自驾环线是什么」「机票的真实性怎么保证」「一家六口带老人该开什么车」——这些问题，没有任何一本书、任何一个 API 能替你回答。它们来自你对这个领域的真实理解、你踩过的真实的坑、你攒下的真实的判断。

正因为它专业，它**抄不走**。别人可以抄你的架构，但抄不走你对这个领域一款款车、一条条路、一个个用户抱怨攒下来的判断。

## 结合点：四个注入口

那么，这两个因子在哪里「相乘」？

答案是第四部分最后一章讲的那个——**四个注入口。** 通用架构提供了口子（prompt、工具、代码、数据），专业能力往里灌内容（原则、细则、规则、事实与判断）。**这就是「通用」和「专业」最具体的结合点**：架构是那副骨架和那几个注入口，专业是灌进去的血肉。

一个专用 Agent 有多「专业」，取决于这个结合的质量——**同样一副通用骨架，灌进去的领域知识越准、越真、越深，长出来的 Agent 就越懂行。**

## 一个关于时代的判断：通用会被抹平

现在讲我认为这一章、乃至整本书最重要的一个判断。

**「通用」这一半，正在被快速地商品化、抹平。** 模型越来越强、越来越便宜；Agent 框架越来越成熟；Claude Code 这样的范式把最佳实践公开了。这意味着——**「通用架构」这个因子，正在变成一个人人都能拿到的东西。** 今天你搭一套 loop + 工具 + 上下文的架构还算有点门槛，但这个门槛在飞快地降低，很快就会变成「谁都能用现成的搭起来」。

而「专业」这一半——领域判断、真实数据、护城河——是**抄不走的、时间友好的**（越攒越深）。它不会因为模型变强而贬值，反而因为「通用能力人人都有」而更凸显。

**所以做专用 Agent，长期的价值和壁垒，正越来越集中到「专业」那一半。** 这是一个方向性的判断：**别指望在「通用架构」上建立差异化**——那部分会被抹平，你今天的领先，明天就是别人的标配。**把差异化，押在「专业」那一半**：最懂这个领域、拿到最真实的数据、攒下最准的判断。

## 这对创作者意味着什么

翻译成行动，就是两句话：

- **通用架构：学最好的范式，别重复发明。** 把它当成一个要熟练掌握、但不必自己创新的工具。省时间、省精力。
- **省下的一切，全押到专业那一半。** 去做那个最懂领域的人、拿到那份最真实的数据、攒下那套最准的判断。**你的护城河，不在通用架构里，在通用架构之外、专业能力之内。**

## 为什么是「×」，不是「+」

最后，为什么这个命题写成乘法「×」，而不是加法「+」？

因为它们是**相乘**的关系，任一个为零，结果就为零：

- **只有通用架构、没有专业能力**——是一个套了领域皮的通用 Agent，它有循环、有工具，但不懂业务，给出的东西经不起这个领域的推敲（就是开篇说的、那个「会说漂亮话但没法直接出发」的通用 Agent）；
- **只有专业能力、没有通用架构**——是一堆领域知识，但没有一个会用它的引擎，是一份死数据、一本没人查的手册。

**两个都要，而且是相乘。** 但既然「通用」那一半正在趋近于一个「人人都有的常数」，那么真正决定你们乘积差异的，就是「专业」那一半有多大。**通用架构让你入场，专业能力决定你能走多远。**
