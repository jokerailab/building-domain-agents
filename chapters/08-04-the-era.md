# 这个时代，属于「懂行的人」

走到这里，从架构、模型、工具，到业务、数据、产品、方法，一整条链路都走过了一遍。最后一章，把视线从「怎么做」抬起来，看向一个更大的问题——**这一切，意味着什么。**

## 回到开篇那个判断

这本书是从一个判断开始的：**这个时代，专用领域的 Agent 会越来越多。** 因为通用 Agent 和 Coding Agent，覆盖不了各行各业的专业场景——它们博学，但不懂你的业务。

走完全书，我更确信这个判断，而且能把它讲得更具体了。**专用 Agent 越来越多，不是因为技术变复杂了，恰恰相反——是因为『通用』那一半，正在变得越来越简单、越来越现成。**

## 为什么是现在

做一个专用 Agent，过去需要你**同时**是两种人：既懂 AI 架构（怎么搭 loop、管上下文、驯服模型），又懂某个行业的业务。这两样都精通的人很少，所以专用 Agent 稀缺。

但现在，天平在变——**「懂 AI 架构」这一半，正在被飞快地商品化。** 模型越来越强、越来越便宜；Agent 的最佳实践被 Claude Code 这样的范式公开、趟平；搭一套像样的骨架，门槛在肉眼可见地降低。

这意味着一件事：**做专用 Agent 的门槛，正在从「既懂 AI 又懂业务」，降到「懂业务、且愿意把现成的 AI 架构搬过来」。** 那个曾经最稀缺的「AI 架构」能力，正在变成一个谁都能拿到的工具。**而一旦工具普及了，胜负手就回到了『谁更懂那个行业』。**

## 谁会做出下一批专用 Agent

所以，做出下一批优秀专用 Agent 的，多半**不是最懂技术的人，而是最懂某个领域、又愿意把它做真、做深的人。**

因为这本书反复证明的那件事——一个专用 Agent 的价值，一大半在「专业」那一半：领域的真实理解、真实的数据、准确的判断、对用户到底要什么的洞察。这些，是**领域里的人**才有的东西。而他们过去缺的那块「AI 架构」，现在有了越来越好的现成答案。

**领域专家 × 现成的通用架构——这会是下一批专用 Agent 的主要形态。** 那个最懂医疗的医生、最懂法律的律师、最懂某类制造的工程师、最懂某种玩法的资深玩家，配上一套他能驾驭的通用 Agent 架构，就能做出一个通用 Agent 永远给不了的东西。

## 各行各业，都值得有一个

顺着这个逻辑往下——**几乎每一个专业领域，都值得有一个「既有 Agent 架构、又懂本行」的专用 Agent。**

医疗、法律、金融、教育、旅行、制造、农业……每一个行业，都有一堆通用 Agent 处理不好的专业判断、一堆只有内行才懂的真实性要求、一堆散落在从业者脑子里、从没被结构化过的经验。**这些空白，就是专用 Agent 的位置。** 通用 Agent 填不了它们，因为它不够懂；而懂的人，现在有了填补它们的工具。

这本书讲的野行 Y——一个自驾行程 Agent、后来的约伴平台——**只是这个更大图景里的一个例子。** 它的价值不在「旅行」这个具体领域，而在它走通的那条路：**把通用的 Agent 架构，和一个具体领域的专业能力，结合起来。** 这条路，可以迁移到任何一个你真正懂的领域。

## 给站在这个时代的你

如果你读到这里，在想「我能不能做一个」，那么这本书最后想给你的，是三句话：

**第一，找一个你真正懂的领域。** 不是「看起来有机会」的领域，是你有真实理解、真实判断、甚至真实数据的领域。你的优势不在 AI，在你比别人更懂的那个行当。

**第二，通用架构，学最好的、别自己发明。** 它是你的入场券，而且这张券越来越便宜。把它当成要熟练使用的工具，别当成要攻克的难关——省下的精力有更重要的去处。

**第三，把一切押在「专业」和「真实」上。** 最懂这个领域、拿到最真实的数据、守住最硬的真实性底线、攒下最准的判断。**这些是抄不走的，是你的护城河，也是这个时代真正稀缺的东西。**

## 尾声

回到最开始。通用 Agent 是一个博学但没有工具、没有数据、没有专业判断的通才；Coding Agent 证明了「通用架构 × 专业领域」这条路走得通。而这本书，是把这条路，走了一遍给你看。

**这个时代，通用能力会越来越像空气——无处不在、也越来越不值钱；而『懂行』，会越来越珍贵。** 属于这个时代的机会，不在于你多会调模型，而在于你有多懂某一个真实的世界，以及你愿不愿意把那份「懂」，用这套方法，认认真真地，做成一个真正有用的东西。

去做那个懂行的人。剩下的，这本书已经讲给你了。
