# 踩过的坑

一套方法论，光有正面的道理不够——更值钱的，常常是那些「**踩过才知道」的教训**。这一章把全书散落的坑，收成一份清单。它们有一个共同点：**看起来对、听起来合理，实则会在某个时刻咬你一口。**

## 关于模型和架构

**别把确定性规则交给模型「自觉」。** 凡是「不能出错、必须每次都执行」的规则（离群坐标丢弃、畸形结构矫正、驾时守卫），焊进代码，别写进 prompt 指望模型每次都记得。模型会漏、会忘、会时对时错——**确定的事，交给确定的代码。**

**别以为模型是可互换的零件。** 强模型和弱模型的能力是有下限的，掉到弱模型时，Agent 会「变蠢」——但这是能力下限，不是你的代码回归了。**分不清「模型变蠢」和「代码坏了」，你会去修一个根本不存在的 bug。** 把「当前是哪个模型」做成看得见的仪表盘。

**别在通用架构上求差异化。** 那部分正在被抹平，你今天的领先明天就是别人的标配。差异化押在专业能力上。

**别为不存在的规模，上重的架构。** 单 worker 够用就别急着上多 worker，配置级能榨的先榨干再谈架构级。**过早的架构复杂度，是从落地那天起就要你天天买单的负债。**

## 关于真实性和数据

**别只靠 prompt 恳求模型「不要编造」。** 靠架构让它**编不了**——把不能错的数据（价格、航班号）放到模型够不着的地方。防编造是架构问题，不是措辞问题。

**别拿真实数据当测试目标。** 你辛苦攒的真实数据（种子内容、判断库）是资产，测试用一次性的假数据。**用真实资产做测试，是在拿护城河冒险。**

**别用假人填充冷启动。** 用真实内容填充「浏览」可以，用伪造的关系填充「社交」不行——用户点了「加入」发现没人的那一刻，信任就崩了。

**别为了「看起来完整」而填一个猜的值。** 拿不准就留空。**一个塞满猜测的数据库，比一个有空缺的更糟**——空缺是诚实的，错误的填充是骗人的。

**别用一套逻辑通吃境内外。** 境内外的差异是系统性的、贯穿全栈的。显式分叉，而且要**贯穿到底**——只分一半（取数分了、展示没分），比不分更糟。

## 关于验证和调试

**别信任何「诊断报告」直接就去改。** AI、分析工具、子任务给的问题清单，对的时候很对、错的时候一样自信——**把每一条当「待验证假设」，逐条实测证实或证伪了再动手。**

**别只验「它正常时的样子」。** 要验「它真实会怎么坏」。那些不报错的失败（静默跳过、悄悄降级、默默阻塞）最阴险，happy-path 的验证根本照不到它们。

**别把「代码看起来对」当成「问题解决了」。** 尤其视觉和地理，最终裁判是渲染出来的像素、地图上的点，不是源码里的数值。**用眼睛看，用真实会触发的输入跑一遍。**

**别忘了排序 bug 可能在模板层。** 后端 sort 对了，模板重新分组渲染照样能把顺序盖掉。**一个现象有多个可能的层，别只查你第一个想到的那层。**

## 关于工具和产品设计

**别按代码模块划分工具，要按模型的「意图颗粒度」划分。** 工具是给模型用的，该匹配「模型脑子里的一个个意图」，不是你的代码结构。

**别把「可参考」的数据伪装成「精确」的。** 参考价就标参考价，估算就说是估算。**假装精确，比诚实地模糊更伤信任**——因为它会在下单那一刻当场穿帮。

**别过早做重的产品化方案。** BYOK 先于支付、免费次数先于精确计量、邀请链接先于全民注册。先用最轻的跑通闭环、验证假设，再上重的。但也别拖到最后——**早做，但做轻。**

**别在反爬上无止境地烧时间。** 你的数据库要能独立服务，抓取只是增量补充。想清楚这点，就不会陷进跟反爬的军备竞赛。

## 一些具体到「会让你怀疑人生」的技术坑

**改了样式没生效？先怀疑缓存，不是你的代码。** CDN 按扩展名缓存、URL 不变，浏览器一直拿旧版——它会完美伪装成「你的改动没生效」，让你在一个不存在的问题上耗半天。给资源地址带上随内容变的版本号。

**单 worker + 异步框架，别在异步接口里跑同步重活。** 一个几十秒的阻塞调用能冻结整个事件循环、让全站一起卡死，而且不报错、本地低负载还难复现。重活丢线程池。

**中文加粗，注意标点边界。** 有些 markdown 渲染下，加粗记号紧贴中文标点会失效、把记号原样漏出来。这类「排版层」的小坑，同样得靠**渲染出来看一眼**才能抓到——又一次印证那条铁律。

## 这些坑的共同点

回看这份清单，会发现它们几乎都指向同一个根源——**「看起来对」和「真的对」之间的那道缝。**

代码看起来会那样走（但实测走了别的路）、这个数据看起来是真的（但没人核实过）、这个报告看起来很专业（但一半是伪问题）、这个架构看起来更先进（但你根本不需要）。**几乎每一个坑，都是在「看起来对」的地方，省掉了「验证一下」的那一步。**

所以如果这份清单只能留一条，那就是全书那条贯穿始终的铁律：**别停在「看起来对」，动手让它变成「我验证过是对的」。** 这一条，能帮你避开上面绝大多数坑。
